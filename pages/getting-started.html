Getting Started

Set up your environment, install the Rialo CDK, and deploy your first on-chain program in minutes.

On this page

  • Install the CDK
  • Your first program
  • Build & deploy

Prerequisites

Before you start you'll need a few standard tools. Rialo programs are written in Rust and compiled to the Rialo VM (RISC-V / SVM-compatible). If you've built on Solana before, the toolchain will feel familiar.

1

Install Rust

Rialo requires Rust stable (≥ 1.75). Install via rustup:

bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
rustup update stable
rustup target add wasm32-unknown-unknown
2

Install the Rialo CLI

The CLI handles project scaffolding, local simulation, and deployments.

bash
cargo install rialo-cli --locked

Verify the install:

bash
rialo --version
# rialo-cli 0.9.x
3

Install Node.js (optional — for client SDKs)

If you want to interact with deployed programs from a TypeScript/JavaScript client, install Node.js ≥ 18.

Install the CDK

The Rialo CDK (rialo-cdk) is the primary Rust library for building on-chain programs. Add it to your Cargo.toml:

toml
[package]
name    = "my-rialo-program"
version = "0.1.0"
edition = "2021"

[dependencies]
rialo-cdk     = "0.9"
rialo-venus-dsl = "0.9"   # async DSL macros — AFTER, EVERY, ON, SEND, START

[lib]
crate-type = ["cdylib", "lib"]
ℹ️
Solana compatibility

Rialo's SVM layer is API-compatible with Solana programs. If you have an existing Anchor project, you can port it using rialo-sol-lang in place of the standard Anchor dependency with minimal changes.

Your first program

Create a new project with the CLI scaffolder:

bash
rialo new counter
cd counter

The generated src/lib.rs shows the minimal structure of a Rialo program using the Venus DSL:

rust
use rialo_cdk::prelude::*;
use rialo_venus_dsl::rialo;

rialo! {
    // Persistent on-chain state
    state {
        count: u64,
        owner: Pubkey,
    }

    // Program entrypoints
    program {
        // Callable by anyone — increments the counter
        fn increment(ctx: Context) {
            state.count += 1;
            emit!(CounterIncremented { count: state.count });
        }

        // Only the owner can reset
        fn reset(ctx: Context) {
            require!(ctx.signer == state.owner, ErrorCode::Unauthorized);
            state.count = 0;
        }
    }
}

// Event definition
#[event]
pub struct CounterIncremented {
    pub count: u64,
}

Key things to notice:

  • The rialo! macro is the entry point for any Venus DSL program.
  • state { } defines persistent data that survives across transactions — no manual account serialization.
  • program { } holds your callable functions and async workflows.
  • emit! fires an on-chain event that external listeners or other programs can react to.

Build & deploy

Build the program for the Rialo VM:

bash
rialo build

Run a local simulation to test before deploying:

bash
rialo localnet start        # spins up a local Rialo node
rialo deploy ./target/deploy/counter.so --url localhost
rialo call counter increment --url localhost
✅
Tip — watch mode

Run rialo localnet watch to automatically redeploy when you save changes during development.

Connect to testnet

The Rialo testnet launched in early 2026. Request a testnet wallet and airdrop tokens to get started:

bash
# Generate a new keypair
rialo keygen --outfile ~/.config/rialo/id.json

# Airdrop testnet tokens
rialo airdrop 2 --url https://testnet.rialo.io

# Deploy to testnet
rialo deploy ./target/deploy/counter.so --url https://testnet.rialo.io
