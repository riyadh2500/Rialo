Why Rialo for RWAs

The RWA market grew 266% in 2025 and is widely projected to reach the trillion-dollar scale. But most current RWA implementations are static — a token that represents an asset, with a human or off-chain bot needed to update its state when real-world events occur.

Rialo makes RWAs dynamic. An asset can:

  • Automatically pay coupons when a timer fires
  • Revalue itself when a price feed crosses a threshold
  • Trigger a margin call when collateral ratios drop
  • Read a corporate action from an external API and update token metadata
  • Cross-chain settle without a bridge

All of this runs at the protocol level. No keeper bot. No oracle subscription. No off-chain scheduler to maintain.

Asset lifecycle

A complete RWA on Rialo goes through four stages, each of which can be automated:

1

Issuance

The asset is verified (via HTTPS calls to registries, credit agencies, or custody providers) and minted as a token. Metadata is stored on-chain with a verified link to the off-chain asset record.

2

Lifecycle management

Venus DSL timers handle recurring events — coupon payments, rent distributions, revaluations — automatically at the protocol level.

3

Trading & transfer

Tokens trade on Rialo's native DEX or via cross-chain flows. Compliance checks (KYC/AML) can be embedded directly in the transfer logic.

4

Settlement & maturity

At maturity or upon trigger conditions, the program automatically settles — burning tokens and distributing proceeds to holders.

Event-driven RWAs

The key differentiator: Rialo RWAs are reactive. They respond to the world in real time without manual intervention or external infrastructure.

rust
rialo! {
    state {
        asset_value:       u64,
        collateral_ratio:  u64,  // in basis points
        liquidation_threshold: u64,
    }

    program {
        // Re-price every hour using native price feeds
        EVERY 1h => update_valuation();

        // React instantly when collateral drops below safe threshold
        ON price_feed::below("ASSET/USD", state.liquidation_threshold)
            => trigger_margin_call();

        fn update_valuation() {
            let spot = price_feed::get("ASSET/USD").unwrap();
            state.asset_value = spot;
            emit!(AssetRevalued { new_value: spot });
        }

        fn trigger_margin_call() {
            // Notify all holders
            SEND notify_holders();
            // Allow 24h for additional collateral before liquidation
            AFTER 24h => check_and_liquidate();
        }

        fn check_and_liquidate() {
            let current = price_feed::get("ASSET/USD").unwrap();
            if current < state.liquidation_threshold {
                // Auto-liquidate and distribute proceeds
                liquidate_and_distribute();
            }
        }
    }
}

Example: tokenized bond

A fully automated corporate bond with semi-annual coupon payments and automatic maturity settlement.

rust
use rialo_cdk::prelude::*;
use rialo_venus_dsl::rialo;

rialo! {
    state {
        issuer:       Pubkey,
        face_value:   u64,
        coupon_rate:  u64,   // basis points, e.g. 500 = 5%
        holders:      Vec<(Pubkey, u64)>,  // (holder, token_amount)
        maturity:     i64,   // unix timestamp
        total_supply: u64,
    }

    program {
        fn issue(ctx: Context, face_value: u64, coupon_bps: u64, maturity_days: u64) {
            state.issuer      = ctx.signer;
            state.face_value  = face_value;
            state.coupon_rate = coupon_bps;
            state.maturity    = Clock::get().unix_timestamp
                                + (maturity_days as i64 * 86400);
            state.total_supply = face_value;

            // Schedule semi-annual coupon payments
            EVERY 182d => pay_coupon();

            // Schedule maturity settlement
            AFTER maturity_days d => settle_maturity();
        }

        fn pay_coupon() {
            let coupon_per_token = state.face_value
                * state.coupon_rate / 10_000
                / (state.total_supply / 1_000_000);

            for (holder, amount) in &state.holders {
                let payout = coupon_per_token * amount / 1_000_000;
                token::transfer(state.issuer, *holder, payout);
                messaging::send(*holder, format!(
                    "Coupon payment of {} received for bond position", payout
                ));
            }
            emit!(CouponPaid { amount: coupon_per_token });
        }

        fn settle_maturity() {
            for (holder, amount) in &state.holders {
                let redemption = state.face_value * amount / state.total_supply;
                token::transfer(state.issuer, *holder, redemption);
            }
            emit!(BondMatured { face_value: state.face_value });
        }
    }
}

Example: dynamic real estate token

A property token that fetches rental income data from a property management API and distributes yields monthly.

rust
rialo! {
    state {
        property_id:   String,
        manager_api:   String,
        holders:       Vec<(Pubkey, u64)>,
        total_shares:  u64,
        last_valuation: u64,
    }

    program {
        // Distribute rental income monthly
        EVERY 30d => distribute_rental_income();

        // Re-appraise property value quarterly
        EVERY 90d => update_appraisal();

        fn distribute_rental_income() {
            // Fetch income data from property manager API
            let resp = https::get(
                format!("{}/income?property={}", state.manager_api, state.property_id)
            ).unwrap();

            let income: RentalIncome = resp.json();
            let net_income = income.gross - income.expenses;

            for (holder, shares) in &state.holders {
                let share = net_income * shares / state.total_shares;
                if share > 0 {
                    token::transfer_stable(*holder, share);
                }
            }
            emit!(RentalDistributed { amount: net_income });
        }

        fn update_appraisal() {
            let resp = https::get(
                format!("{}/valuation?property={}", state.manager_api, state.property_id)
            ).unwrap();
            let appraisal: PropertyValuation = resp.json();
            state.last_valuation = appraisal.value_usd;
            emit!(PropertyRevalued { value: appraisal.value_usd });
        }
    }
}
