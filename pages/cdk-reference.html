CDK Reference

Complete reference for the Rialo Rust crates — rialo-cdk, rialo-venus-dsl, rialo-sol-lang, and supporting libraries.

Crate overview

CrateVersionPurpose
rialo-cdk 0.9.x Primary on-chain program library. Targets both native (tokio async) and wasm32.
rialo-venus-dsl 0.9.x Async workflow DSL — rialo! macro, AFTER / EVERY / ON / SEND / START.
rialo-sol-lang 0.9.x Solana Sealevel eDSL (Anchor-compatible). For porting or writing SVM-layer programs.
rialo-s-program 0.9.x Low-level on-chain program crate (Solana program layer). Use when you need direct SVM access.
rialo-s-sdk 0.9.x Client-side Rust SDK. Use this to interact with deployed programs from off-chain code.
rialo-s-feature-set 0.9.x Feature flags and capability detection for the Rialo runtime.

rialo-cdk

The primary library for building on-chain programs. It re-exports everything you need and provides the core abstractions: context, token operations, events, accounts, and native primitives.

Prelude

rust
use rialo_cdk::prelude::*;
// Brings into scope: Context, Pubkey, Result, require!, emit!,
// Clock, token, https, price_feed, messaging, random, schedule

Core types

TypeDescription
ContextTransaction context — ctx.signer, ctx.accounts, ctx.auth_token
Pubkey32-byte on-chain address
Result<T>Rialo result type, maps to ProgramResult
DurationTime span — Duration::seconds(n), Duration::days(n)
ClockOn-chain clock — Clock::get().unix_timestamp

Macros

MacroDescription
require!(cond, err)Assert condition or return error. Equivalent to if !cond { return Err(err) }
emit!(event)Emit a typed on-chain event from a struct decorated with #[event]
msg!(fmt, ...)Log a message to the transaction log (debug use)

rialo-venus-dsl

Provides the rialo! proc-macro and the Venus async DSL keywords. See the Venus DSL reference for full usage documentation.

Macro entry point

rust
use rialo_venus_dsl::rialo;

rialo! {
    state { /* fields */ }
    program { /* functions + async ops */ }
}

Async operation syntax

KeywordSyntaxDescription
AFTERAFTER <duration> => fn()Run once after delay
EVERYEVERY <duration> => fn()Run repeatedly on interval
ONON <condition> => fn()Run when event/condition fires
SENDSEND fn(args)Non-blocking async message
STARTSTART WorkflowName { fields }Spawn new workflow instance

rialo-sol-lang

An Anchor-compatible eDSL for writing Solana Sealevel programs on Rialo. If you're familiar with Anchor, this will feel identical — it's a drop-in replacement that targets the Rialo SVM layer.

rust
use rialo_sol_lang::prelude::*;

#[program]
pub mod my_program {
    use super::*;

    pub fn initialize(ctx: Context, data: u64) -> Result<()> {
        ctx.accounts.my_account.data = data;
        Ok(())
    }
}

#[derive(Accounts)]
pub struct Initialize<'info> {
    #[account(init, payer = user, space = 8 + 8)]
    pub my_account: Account<'info, MyAccount>,
    #[account(mut)]
    pub user: Signer<'info>,
    pub system_program: Program<'info, System>,
}

#[account]
pub struct MyAccount {
    pub data: u64,
}

rialo-s-sdk (client)

Use rialo-s-sdk to interact with deployed programs from client-side Rust code (off-chain, native async).

toml
# Client binary Cargo.toml
[dependencies]
rialo-s-sdk = "0.9"
tokio = { version = "1", features = ["full"] }
rust
use rialo_s_sdk::{
    client::RpcClient,
    signature::{Keypair, Signer},
    transaction::Transaction,
};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let client = RpcClient::new("https://testnet.rialo.io");
    let payer  = Keypair::read_from_file("~/.config/rialo/id.json")?;

    // Call a program instruction
    let ix = my_program::instruction::increment(
        &my_program::ID,
        &payer.pubkey(),
    );

    let tx = Transaction::new_signed_with_payer(
        &[ix],
        Some(&payer.pubkey()),
        &[&payer],
        client.get_latest_blockhash().await?,
    );

    let sig = client.send_and_confirm_transaction(&tx).await?;
    println!("Transaction: {}", sig);
    Ok(())
}
