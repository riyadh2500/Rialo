Frequently Asked Questions

Common questions about Rialo — the network, development, testnet, and what's coming next.

General

What is Rialo?

Rialo is a real-world blockchain built by Subzero Labs. It's a decentralized, high-performance programmable network that natively supports HTTP calls, real-time price feeds, timers, cross-chain data, and privacy features — without external oracles, bridges, or middleware. It calls itself "not a layer one" because it's designed to work more like a programmable operating system than a traditional isolated blockchain.

Who built Rialo?

Rialo is built by Subzero Labs, led by Ade Adepoju and Lu Zhang — both early engineers at Mysten Labs who played foundational roles building the Sui network. The $20M seed round was led by Pantera Capital, with Variant, Coinbase Ventures, and Susquehanna also participating.

How is Rialo different from Solana or Ethereum?

On Solana or Ethereum, you must assemble a stack of third-party services to build real-world apps: Chainlink/Pyth for price data, Gelato/Clockwork for automation, Wormhole for cross-chain, and various privacy tools. Rialo builds all of these as native protocol primitives — same trust assumptions as the chain itself, stable costs, no extra vendors. The Venus DSL also makes async, event-driven programs feel much more like modern application code.

Development

What language do I use to write Rialo programs?

Rust. Programs use the rialo-cdk library and the rialo-venus-dsl macro crate. The toolchain is intentionally close to Solana/Anchor — if you've built on Solana, you can port your program with minimal changes using rialo-sol-lang as a drop-in Anchor replacement.

What is the Venus DSL?

Venus is Rialo's embedded DSL for writing async, event-driven on-chain programs. It adds five keywords:

  • AFTER <duration> — run once after a delay
  • EVERY <duration> — run repeatedly on an interval
  • ON <condition> — react when an event fires
  • SEND <fn>() — non-blocking async message
  • START <workflow> — spawn a new workflow instance

These are protocol-level primitives — no keeper bot, no hosted scheduler, no third-party service required.

Do I need an oracle for price data?

No. Call price_feed::get("BTC/USD") directly in your program. Native feeds cover all major trading pairs with stable, predictable costs. No Chainlink subscription, no Pyth program, no extra trust assumption.

Can I port my existing Solana / Anchor program?

Yes. Replace anchor-lang with rialo-sol-lang in your Cargo.toml and update the deploy target. The API is compatible. After porting, your program gains access to all of Rialo's native primitives immediately.

How do I run recurring tasks without a bot?

Use EVERY 24h => my_task(); inside your program { } block. The timer is registered with the Rialo protocol at deploy time and executed by the network. Nothing to host. Nothing to monitor.

Network & Status

Is the testnet live?

Yes. The Rialo testnet launched in early 2026. Access is currently gated via the early-access builder program. Apply here →

When is mainnet?

Mainnet is targeted for 2026. No specific date has been announced publicly. Follow @RialoHQ and the Rialo blog for updates.

Is there a token or airdrop?

Rialo has not confirmed a token or airdrop. Any official announcement will come from rialo.io, the official Discord, or @RialoHQ only. Do not trust unofficial sources.

What VMs does Rialo support?

Currently available: REX (RISC-V) — Rialo's native runtime, ZK-friendly and TEE-integrated — and SVM (Solana VM), fully Anchor-compatible.

Planned: EVM (Solidity support) and MoveVM (Sui/Aptos developer support).
